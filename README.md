# Paid Bounty Triage Agent

Paid Bounty Triage Agent is a minimal paid/callable AI-agent demo for the CROO Agent Hackathon. A requester submits a bounty or hackathon URL, pays for a triage call, receives a structured go/no-go report, and gets a settlement receipt.

The current demo uses mock settlement so it can be reviewed without private keys. The adapter boundary is explicit enough to replace with real CROO CAP SDK calls.

## What It Shows

- Callable surface: CLI `call` and HTTP `POST /call`.
- Paid service envelope: requester, provider, price in USDC, payer wallet, provider wallet.
- Agent output: priority, token estimate, cash probability, recommended track, risks, and delivery checklist.
- Settlement loop: mock CROO/CAP lifecycle events and `receipt.json`.
- Judge artifacts: `agent_call.json`, `triage_report.md`, `settlement_request.json`, `receipt.json`, `summary.html`, and `demo.log`.

## Quick Demo

From this directory:

```bash
python3 paid_bounty_agent.py call \
  --url https://dorahacks.io/hackathon/croo-hackathon \
  --title "CROO Agent Hackathon" \
  --price-usdc 0.25 \
  --run-id latest-cli-demo
```

Then inspect:

```bash
cat artifacts/latest-cli-demo/triage_report.md
cat artifacts/latest-cli-demo/receipt.json
open artifacts/latest-cli-demo/summary.html
```

## Callable HTTP Demo

Terminal 1:

```bash
python3 paid_bounty_agent.py serve --host 127.0.0.1 --port 8787
```

If `8787` is already in use, switch both commands to another local port, for example `8791`.

Terminal 2:

```bash
curl -sS -X POST http://127.0.0.1:8787/call \
  -H 'content-type: application/json' \
  --data @examples/request.json
```

The response includes the run directory and receipt summary. The full artifact set is written to `artifacts/latest-http-demo`.

## Real CROO CAP Replacement

Mock mode is implemented by `MockSettlementAdapter`. Real mode should replace it with the CROO SDK lifecycle:

- Python SDK: `pip install croo-sdk`.
- Node SDK: `npm install @croo-network/sdk`.
- Environment: `CROO_API_URL=https://api.croo.network`, `CROO_WS_URL=wss://api.croo.network/ws`, `CROO_SDK_KEY=croo_sk_...`.
- Requester flow: `negotiate_order`, then `pay_order`.
- Provider flow: `accept_negotiation`, then `deliver_order`.
- Delivery retrieval: `get_delivery`.
- Default RPC in SDK docs: `https://mainnet.base.org`.
- Payment asset: USDC deposited to the agent AA wallet shown in the CROO Dashboard.

Private keys, seed phrases, cookies, and wallet sessions must not be passed to this demo. Real wallet signing is a manual confirmation gate before payment.
